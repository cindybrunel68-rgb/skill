# Workflow LinkedIn MDF

## Objectif
Créer un flux simple pour produire des posts LinkedIn en cohérence avec la voix de Cindy, puis les préparer pour une publication ultérieure, de façon progressive et sécurisée.

## Principe
Le workflow fonctionne en 3 couches :
1. Génération automatique de drafts
2. Validation humaine
3. Publication programmée ou manuelle

Aucune publication automatique n’est faite tant que le contenu n’a pas été validé.

## Entrées nécessaires
- Sujet ou idée de post
- Source de contenu : vidéo, notes, article, réflexion, événement
- Public cible : frontaliers, patrons, prospects, etc.
- Objectif du post : informer, rassurer, générer des rendez-vous, créer de la confiance
- Date de publication souhaitée

## Étapes du workflow

### 1. Capture de l’idée
Récupérer une source de contenu :
- une vidéo
- un échange
- un constat métier
- un article ou un sujet d’actualité

Créer un enregistrement simple avec :
- titre du sujet
- message principal
- angle de contenu
- appel à l’action ou question finale

### 2. Extraction des thèmes
Transformer la source en 3 à 5 idées possibles :
- une idée forte
- une idée pédagogique
- une idée de preuve / expérience
- une idée de réflexion
- une idée de conversion

### 3. Génération des drafts
Pour chaque idée, produire :
- 1 post court
- 1 post moyen
- 1 version plus narrative

Le contenu doit respecter :
- la voix MDF
- le vouvoiement
- le ton rassurant et expert
- la simplicité de langage
- l’absence de jargon inutile

### 4. Vérification de cohérence
Avant toute publication, vérifier :
- le post sonne comme Cindy
- il est clair et lisible
- il n’est ni trop générique ni trop robotique
- il a un angle utile pour la cible
- il finit sur une ouverture ou une question naturelle

### 5. Validation humaine
Mettre le contenu dans une file d’attente de validation :
- accepté
- à retravailler
- à rejeter

À ce stade, l’IA propose, l’humain décide.

### 6. Préparation de publication
Une fois validé, préparer :
- le texte final
- la version courte si nécessaire
- les hashtags utiles
- la date de publication
- l’image ou le visuel associé si besoin

### 7. Publication
Deux modes possibles :
- publication manuelle
- publication programmée plus tard

### 8. Suivi de performance
Après publication, noter :
- nombre de vues
- réactions
- commentaires
- liens ou messages reçus

Cela sert à améliorer les prochains posts.

## Structure de stockage recommandée
Créer un dossier de travail par période :
- .tmp/linkedin/
  - ideas/
  - drafts/
  - approved/
  - scheduled/
  - published/

## Format de sortie attendu
Pour chaque post, garder :
- titre
- sujet
- angle
- version finale
- statut
- date de publication

## Règle de sécurité
- aucune publication automatique sans validation
- aucun contenu sensible sans relecture
- le contenu doit rester aligné avec la marque et la voix

## Version simple à utiliser dès maintenant
1. Soumettre une idée
2. L’IA génère 3 versions
3. Vous choisissez la meilleure
4. Vous publiez ou vous programmez

## Version automatisée future
Une fois le flux validé, on peut ajouter :
- une file de contenu hebdomadaire
- une génération automatique de posts à partir de vidéos ou d’articles
- une programmation de publication
- un suivi de performance
